# Loop design context

## Users

TIST staff who run the company day-to-day. They send emails, use WhatsApp, never opened Linear or Jira. They want to hand each other small jobs without learning a new tool.

The product lives on a laptop during the workday. Users glance at it during meetings. The work is small: send invoices, design the welcome poster, order whiteboard markers. A quarter-long initiative does not belong here.

## Brand personality

Three words: **warm**, **playful**, **calm**.

Loop should feel like a friendly coworker. The TIST interviewer remembered Vishal's portfolio and resume builder because they both used sound design. Loop uses sound the same way: a small reward for finishing something.

## Aesthetic direction

Loop borrows from three apps without copying any.

- **Todoist** for the warm task surface: one assignee per task, Quick Add as the main interaction, a ding when you finish something.
- **Linear** for the sidebar shape: quiet chrome, three collapsible sections (Workspace, Projects, Try), keyboard speed.
- **ElevenLabs** for the modal style: an icon tile and title at the top, an X to close, soft layered shadows, generous breathing room.

Light mode only. The background is a warm cream, lighter than Linear's neutral, warmer than white. Loop red appears in four places: the brand tile, primary buttons, the priority-1 flag, and the focus ring. The rest of the chrome stays neutral.

What Loop should never resemble: Jira, Asana, any tool that needs a training session. Karma badges, streak fireworks, and "enterprise feel" all stay out.

## Design principles

1. **Quiet chrome, warm surface**. Linear restraint in navigation and headers. Todoist warmth in the task canvas.
2. **One signature red**. Use it for the four cases above. If a third red shows up in a screenshot, one of them is wrong.
3. **Apply the UX laws Vishal cares about**: Fitts (40px hit zones), Hick (curate choices), Miller (chunk lists by day or section), Doherty (sub-400ms feedback, optimistic UI everywhere), Postel (forgiving date and text parsing).
4. **Sound is product**. Five synthesized tones cover *added*, *assigned to me*, *completed*, *streak*, and *error*. A mute toggle lives in the sidebar footer.
5. **Defer everything to the assign-task flow**. Someone types a task in Quick Add and picks a teammate. Supabase realtime fires. The teammate's tab updates without a refresh, a chime plays, a toast appears. Every other surface exists to make this moment land.
6. **No jargon**. The product never says "issue", "ticket", "epic", or "sprint". It only says "task".

## Tech constraints

- Next.js 16 (Turbopack default), Tailwind v4, shadcn (Base UI under the hood), React 19.
- Supabase for auth, Postgres, and realtime via `postgres_changes`.
- Phosphor icons at 16px regular weight, with `fill` for priority flags. Imported through `components/icons.ts` so server pages can use them.
- `motion` v12 for animations.
- Web Audio API for sounds. No asset files.
- Light mode only. Dark-mode tokens exist in `globals.css` but no toggle wires them up yet.
