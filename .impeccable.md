# Loop design context

## Users

Internal team at one company — small-to-medium (5–30 people) — who run
the day-to-day operations. They send emails, use WhatsApp, never opened
Linear or Jira. They want to hand each other small jobs without learning
a new tool.

The product lives in a browser tab from morning standup to evening
shutdown. Users glance at it during meetings, triage from their phone on
the bus, and edit at their desk. The work is small: send invoices, design
the welcome poster, order whiteboard markers. A quarter-long initiative
does not belong here.

**Implications:**
- Density matters — they live in this. More tasks per screen, not less.
- Mobile is for triage (mark done, see status, comment). Heavy editing
  belongs on desktop.
- Onboarding flourish carries less weight than command-velocity:
  keyboard shortcuts, optimistic updates, no spinners for local work.

## Brand personality

Three words: **playful**, **warm**, **a little weird**.

Loop should feel like a friendly coworker, not a productivity sergeant.
The TIST interviewer remembered Vishal's portfolio and resume builder
because they both used sound design. Loop uses sound the same way: a
small reward for finishing something.

The "a little weird" matters. A teammate scrolling past a screenshot
should know it's Loop without reading the URL — because of the pink, the
Switzer, the friendly copy, the chime. Sanding these off makes it
generic.

## Aesthetic direction

Loop borrows from three apps without copying any:

- **Todoist** for the warm task surface: one primary assignee per task,
  Quick Add as the main interaction, a chime when you finish something.
- **Linear** for the sidebar shape: quiet chrome, collapsible sections,
  keyboard speed. Not Linear's color or hairline grid — just its
  restraint in navigation chrome.
- **ElevenLabs** for the modal style: an icon tile and title at the top,
  an X to close, soft layered shadows, generous breathing room.

### Color

Cool blue-grey neutrals with a hot Y2K pink primary
(`oklch(0.62 0.24 348)`). The pink appears in four places: the brand
tile, primary buttons, the focus ring, and the active-state pill. The
priority-1 flag and "overdue" tone are **rose**, not pink — they're a
semantic accent, not the brand color (this is the change from the
original red-everywhere model).

Semantic accents: rose (overdue, P1, destructive), amber (today, P2),
emerald (completion, low-priority done), muted-foreground (everything
else). OKLCH throughout — perceptually uniform, theme-symmetric.

### Theme

Both light and dark are first-class. Dark is the daily-driver in the
team's actual usage (most screenshots are dark mode), but light must
hold its own — the internal team includes light-mode loyalists. Both
themes get the same color palette via the dark/light token swap in
`globals.css`.

### Typography

**Switzer** (Fontshare) for everything. Higher x-height than Geist/Inter,
which matters at the 11–13px sizes the dense UI uses (chip text, sidebar
items, drawer details). One family for now — no display face yet. If we
add one, it lands on marketing surfaces, not the canvas.

### Iconography

**Phosphor** at the regular weight, with `fill` for priority flags and
active states. Already in use; matches the warm/playful voice better
than Lucide's sharper geometry. Imported through `components/icons.ts`
so server pages can use them.

### Motion

Spring-physics on personality moments (bottom-sheet dismiss, FAB tap,
drawer drag, active-tab pill on mobile bottom nav). Exponential ease-out
(`cubic-bezier(0.32, 0.72, 0, 1)`) for state changes and entrances.
`prefers-reduced-motion` respected via `<MotionConfig reducedMotion="user">`
at the app shell. Never bounce or elastic — those read as tacky.

## Anti-references

What Loop should never resemble:

- **Jira / Asana enterprise.** Heavy chrome, nested dropdowns, "we have
  47 fields per task." The reason Loop exists. Every time a control gets
  added, ask: is this Jira leaking back in?
- **Generic shadcn template.** The "I just shipped from v0" look —
  default radii, default shadows, gray cards stacked uniformly, no point
  of view. Identifiable as AI-generated. Every Loop surface should be
  unmistakable in a screenshot.
- Karma badges, streak fireworks, productivity-arms-race aesthetics.
  Joy is in the chime and the pink, not in gamification.

## Design principles

1. **Joy is the differentiator, not a flourish.** The pink, the chime,
   the spring-back drawer, the conversational copy — these are not
   ornaments. They are the reason Loop wins over a sterile alternative.
   Never apologize for them.
2. **Density respects the user.** Internal team lives in this all day.
   More tasks per screen > more whitespace per task. Resist the urge to
   "give it room to breathe" until it looks like a Dribbble shot with
   three cards.
3. **One signature pink.** The hot Y2K pink is the brand. Use it for the
   four cases above (brand tile, primary buttons, focus ring,
   active-state pill). If a fifth pink shows up in a screenshot, one of
   them is wrong. Semantic colors (rose/amber/emerald) live alongside,
   not as alternatives.
4. **Apply the UX laws that matter here**: Fitts (44pt mobile touch
   targets, 32px+ desktop), Hick (curate choices in pickers), Miller
   (chunk lists by Today/Upcoming/Later), Doherty (sub-400ms feedback,
   optimistic UI everywhere), Postel (forgiving date and text parsing in
   Quick Add).
5. **Sound is product.** Synthesized tones cover *added*, *assigned to
   me*, *completed*, *streak*, and *error*. A mute toggle lives in the
   sidebar footer. Sound is on by default — the chime is the reward.
6. **Defer everything to the assign-task flow.** Someone types a task in
   Quick Add and picks a teammate. Supabase realtime fires. The
   teammate's tab updates without a refresh, a chime plays, a toast
   appears. Every other surface exists to make this moment land.
7. **No jargon.** The product never says "issue", "ticket", "epic", or
   "sprint". It only says "task". Empty states are friendly ("All caught
   up", not "0 items"); errors are conversational, never stack-trace-y.
8. **Identifiable in one screenshot.** Pink + Switzer + chip tones +
   friendly copy carry the brand together. A teammate should know it's
   Loop without reading the URL.

## Tech constraints

- Next.js 16 (Turbopack default), Tailwind v4, shadcn (Base UI under the
  hood), React 19.
- Supabase for auth, Postgres, and realtime via `postgres_changes`.
- Phosphor icons through `components/icons.ts`.
- `motion` v12 for animations.
- Web Audio API for sounds. No asset files.
- Light + dark mode, with token swap in `globals.css`. Theme toggle in
  the sidebar footer.
- Switzer loaded from Fontshare via `<link>` in `app/layout.tsx`. System
  fallbacks in `--font-sans` cover the brief FOUT window.
